1. File Management

1. The Linux File System

The file system is a method of consistently naming objects and organizing them in an orderly manner. Some of the operating system like windows organize the file system in drives. The Linux file system is a directory tree that is consistent with each Linux distribution. This directory tree begins with the "/" (root) and then expands under that position like an upside down tree. Each directory under "/" has a specific purpose whether it is to store system files, constantly changing log files, device drivers or files created by users on the system.

A home directory is a file system sub-directory on a multi-user operating system containing files for a given user of the system. A user's home directory is intended to contain that user's files; including text documents, music, pictures or videos, etc. /home/aditi refers to the home directory of the user aditi, ~ is equivalent to the /home/aditi which is the current users home directory.

The file system also contains a method of securing files, directories and objects in the file system. This is represented by the file permissions that are explained in the experiment File Permissions

2. The Command Line Prompt

The commands that will be described are meant to be executed from a command line interface. The command line prompt that shows up in the terminal window contains useful information. It has the form:

aditi@mylaptop:~/Documents$

There are three parts in the command line prompt, separated by @ : and $

The first part before the '@' shows the user who is logged in at that moment. In the above example it is the user aditi. The second part between the @ and the : shows the name of the computer that you are working on. In the above example it is mylaptop. The third part, between the : and the $ shows the current directory that you are working on. In the above example it is ~/Documents (~ is a shortcut for the userâs home directory; ~/Documents is equivalent to /home/aditi/Documents). When you change the current directory, then they can find the third part of the command line prompt changing.

3. Viewing the contents of files and directories

Viewing the contents of files and directories using three commands, namely pwd, ls and cat.

a. pwd command

At any time, in order to see the current directory you can type pwd(Present Working Directory).

aditi@mylaptop:~/Documents$ pwd
 /home/aditi/Documents

b. ls command

To list the contents of a directory, the ls command is used. When it is executed with no arguments, it lists the contents of the current working directory.

The path of a file is the location of a file in the filesystem. There are two ways of specifying the path of a file: relative path and absolute path. Relative path is the location of a file relative to your current location in the filesystem. Absolute path is the location of the file with respect to the root directory. It is independent of the current working directory.

Example1 : Consider the above directory tree (Figure 1)from which we want to list the contents of the folder my_project. Consider our present working directory is /home/aditi/Documents. Doing "ls my_project" would list the contents of folder my_project, we are using the relative path here as we are listing from the current working directory. If we use "ls /home/aditi/Documents/my_project" from the current working directory, this is considered as the absolute path as we are specifying from "/".

aditi@mylaptop:~/Documents$ ls my_project

In the above example, the relative path of the directory has been used.

aditi@mylaptop:~/Documents$ ls /home/aditi/Documents/my_project
 vlabs_ui

In the above example, the absolute path of the directory has been used. As we can see, the two paths are equivalent.

Referring to the above Figure:1 and assuming our present working directory is /home/aditi/Documents. "." refers to the current directory i.e. Documents and ".." refers to the parent directory(of the current directory) in our example it is aditi(home directory).

"ls ." will display all the contents of the current directory

aditi@mylaptop:~/Documents$ ls .
all_about_console.txt  example.txt  my_project

"ls .." will display all the contents of the parent directory

aditi@mylaptop:~/Documents$ ls ..
Desktop Documents Downloads Music Pictures

The most used options for the ls command are l, a.

Table 1: Common parameters for "ls" command
Option Purpose
lDisplays the detailed view of the contents of the folder
aAlso displays hidden contents of the folder
 
aditi@mylaptop:~/Documents$ ls -l
total 4
-rw-r--r--  1 aditi students 0    2010-08-20 22:22 example.txt
-rw-r--r--  1 aditi students 0    2010-08-20 22:22 all_about_console.txt
drwxr-xr-x  2 aditi students 4096 2010-08-02 14:03 my_project

Here, in addition to the name of each file, the file type, file mode bits, number of hard links, owner name, group name, size, and timestamp, normally the modification time are printed.

The file mode bits contains details regarding the properties of the file. It has 10 fields. The first field refers to the file type. 'd' means the file is a directory, 'l' means it is a link, '-' means it is a regular file. The next nine fields are the permissions of the owner, group and other, respectively. 'r', 'w' and 'x' in a field means 'read', 'write' and 'execute' permissions, respectively. If a '-' is printed in a field, it means that particular permission is not present for the corresponding user, group or other. For detailed explanation go through File Permissions 

aditi@mylaptop:~/Documents$ ls -a
.  ..  all_about_console.txt  example.txt  my_project .virtual-lab.txt

In the above example .virtual-lab.txt was the hidden file, which was displayed when ls was used with -a option.

. and .. refer to current folder and parent folder, respectively.

Whenever you need help about commands in Linux, you can read the man pages, which contain detailed information about commands and their options. To know more about the ls command, see the man page, type: 

aditi@mylaptop:~/Documents$ man ls

Information about the command will appear on the screen. You can move up and down using the arrow keys. In order to exit from the man page type key "q".

c. cat command

cat command displays the contents of the file passed as argument. If multiple files are passed, it prints the concatenated content of all the files.

aditi@mylaptop:~/Documents$ cat virtual-lab.txt
Welcome to Linux Lab!

In the above example cat displays content of the file virtual-lab.txt.

Type the clear command to clear your screen.

4. Moving through directories:

a. cd command

The command cd (change directory) allows us to move through directory trees. It has one parameter: target directory path. If no parameter is mentioned it will go to the current users home directory. If the parameter ".." is used, it will take you to previous directory(parent directory).

aditi@mylaptop:~/Documents$ cd my_project
aditi@mylaptop:~/Documents/my_project$ pwd
/home/aditi/Documents/my_project

aditi@mylaptop:~/Documents/my_project$ cd ..
/home/aditi/Documents

aditi@mylaptop:~Documents/my_project$ cd 

aditi@mylaptop:~$ pwd
/home/aditi

5. Creating files and folders:

b. touch command

It creates an empty file directly from the command line. In order to do it, type:

aditi@mylaptop:~/Documents$ ls
all_about_console.txt example.txt my_project
aditi@mylaptop:~/Documents$ touch newfile.txt
aditi@mylaptop:~/Documents$ ls 
all_about_console.txt  example.txt my_project newfile.txt

An empty file with the specified name will be created. If a file with that name already exists on disk, it will remain there and moment of last update is changed in the properties of the file.

c. mkdir command

The mkdir command takes one or more parameters and creates the folders corresponding to the parameters. 

aditi@mylaptop:~/Documents$ ls 
all_about_console.txt  example.txt  my_project newfile.txt 
aditi@mylaptop:~/Documents$ mkdir new_folder
aditi@mylaptop:~/Documents$ ls 
all_about_console.txt  example.txt  my_project newfile.txt new_folder

6. Copying, Moving and removing files and folders:

a. cp command

cp command is used to create a copy of files. It can be invoked with two arguments, SOURCE and DEST in which case it copies SOURCE to DEST.

 aditi@mylaptop:~/Documents$ ls
all_about_console.txt  example.txt  my_project new_folder

aditi@mylaptop:~/Documents$ cd my_project
aditi@mylaptop:~/Documents$ ls
vlabs_ui 

aditi@mylaptop:~/Documents$ cp example.txt my_project/example_copy.txt
aditi@mylaptop:~/Documents$ ls my_project
vlabs_ui  example_copy.txt

For copying directories, the -r (recursive) option is used.

aditi@mylaptop:~Documents$ ls my_project
vlabs_ui example_copy.txt
aditi@mylaptop:~/Documents$ cp -r new_folder my_project/new_folder_copy 
aditi@mylaptop:~/Documents$ cd my_project
aditi@mylaptop:~/Documents$ ls
vlabs_ui example_copy.txt new_folder_copy

b. mv command

The mv command is used to change the location of directories and files or renaming them. Parameters are taken in a similar way as the cp command.

aditi@mylaptop:~/Documents$ ls
vlabs_ui example_copy.txt new_folder_copy
 
 aditi@mylaptop:~/Documents$ mv example.txt my_project/example_move.txt

 aditi@mylaptop:~/Documents$ ls my_project
 vlabs_ui  example_copy.txt new_folder_copy example_move.txt

c. rm command

rm command removes files. It can be used with the -r option to remove directories.

aditi@mylaptop:~/Documents$ ls 
all_about_console.txt  example.txt  my_project  newfile.txt  new_folder 

aditi@mylaptop:~/Documents$ rm example.txt

aditi@mylaptop:~/Documents$ rm -r new_folder

aditi@mylaptop:~/Documents$ ls
all_about_console.txt  my_project  newfile.txt

d. rmdir command

rmdir command removes empty directories.

aditi@mylaptop:~/Documents$ mkdir new_folder
aditi@mylaptop:~/Documents$ ls
all_about_console.txt my_project newfile.txt  new_folder
aditi@mylaptop:~/Documents$ rmdir new_folder
aditi@mylaptop:~/Documents$ ls
all_about_console.txt  my_project  newfile.txt
