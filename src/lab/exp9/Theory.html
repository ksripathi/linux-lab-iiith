9. Basic Networking

What is an IP address?

An IP address is a way to identify a network device, e.g. a computer, on a network. In order to connect to a computer, you need to know its IP address.

A computer can have multiple IP addresses e.g. one for each interface it has. An interface can be wired or wireless.

How to find this machine's IP address?

Running the command "/sbin/ifconfig" will display the IP addresses of each interface on the machine.

The example below shows that the machine has three interfaces: eth0, lo and wlan0. (Note: the naming of the interfaces varies with different linux distributions, so you may see different interface names on your machine.)

  • "eth0" is the wired interface (eth stands for ethernet). The machine's IP address for this interface is 10.9.24.110.
  • "lo" interface is a local interface i.e. it cannot be used for communicating across the network. This interface is present on all computers. It's IP address is always 127.0.0.1.
  • "wlan0" is the wireless interface. It's IP address is 10.9.24.231.
  • This machine can be contacted via either of the two IP addresses 10.9.24.110 and 10.9.24.231.

How to login to another remote machine

Use ssh (secure shell) to remotely connect to another machine. Type "ssh username@<ip address>" or "ssh username@<hostname>" at the command prompt. 

aditi@mylaptop:~$ ssh shyam@127.0.0.1                                                                               
The authenticity of host '127.0.0.1 (127.0.0.1)' can't be established.
RSA key fingerprint is 28:07:e5:0b:e0:bb:06:65:a6:f3:19:24:bf:22:69:22.
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added '10.9.24.22' (RSA) to the list of known hosts.

You will get the message above if you are connecting to a machine for the first time. Answer "yes" to the question above. 

shyam@127.0.0.1's password:
Permission denied, please try again.
shyam@127.0.0.1's password:

If you enter incorrect password, you will get the above message and prompted to re-enter password.

After you enter the correct password, you will get a message and the shell prompt. The message content depends upon the system administrator and can be modified. Below you can see the shell prompt after the default linux message. 

Linux shyamslaptop 3.2.0-2-amd64 #1 SMP Mon May 21 17:45:41 UTC 2012 x86_64
The programs included with the Debian GNU/Linux system are free software;
the exact distribution terms for each program are described in the
individual files in /usr/share/doc/*/copyright.
Debian GNU/Linux comes with ABSOLUTELY NO WARRANTY, to the extent
permitted by applicable law.
Last login: Tue Jan  8 05:07:49 2013 from 10.9.24.11 
shyam@shyamslaptop:~$

At this stage, you can work in the remote machine as if it was a local machine. After you are done, you can type "exit" to log out of the remote machine. 

shyam@shyamslaptop:~$ exit
logout
Connection to 127.0.0.1 closed.
aditi@mylaptop:~$

You need to know the password on the remote machine in order to connect. Anyone with the userid and password can connect, hence it is important to have a good password on your machine. You can change your password using "passwd" command, as show below. 

aditi@mylaptop:~$ 
(current) Password: 
New password:
Retype new password:
passwd: password updated successfully
aditi@mylaptop:~$

How to copy files from one machine to another machine?

Use "scp" (secure copy) to copy files from one machine to another. 

aditi@mylaptop:~$ scp marks.txt shyam@127.0.0.1:/home/shyam/guestfiles
shyam@127.0.0.1's password:
marks.txt                                          100%     127    10KB/s    00:00
aditi@mylaptop:~$

How to synchronize two files or directories on one machine to another machine

Synchronizing two locations means having the same content (files, directories) on both the locations. You can use "rsync" to synchronize files and folders across different directories in the same computer, or across different computers.

If you want to synchronize two directories with small content (a few kilobytes or a few megabytes), you can just do an "scp" (as explained above) and copy the files from the source to target. But if you need to synchronize bigger directories (several megabytes or even gigabytes), copy will not be effective as scp copies all the content from scratch and will take a lot of time.

rsync only copyies the changes that were done since last sync. If both the locations already have the same content, rsync will not do anything. rsync has the additional benefit that it can retain the file permissions and other attributes across the locations.

Example 1: Synchronize contents of lectures folder. The below command syncs all the files (but not subdirectories) in /home/aditi/lectures directory with lectures folder in Shyam's home directory. Using the "-v" (verbose) option, you can see the files being synchronized. 

rsync -v /home/aditi/lectures/ shyam@127.0.0.1:~/lectures/

Example 2: Synchronize all C programs from present working directory to /opt/cprograms/ directory, retaining the last modified timestamps. Using the "-t" option enables synchronization of timestamps as well (date and time when the file was last modified). 

rsync -t *.c shyam@127.0.0.1:/opt/cprograms/

Example 3: Synchronize all contents of "readings" directory (including subdirectories) to "readings" directory in Shyam's home directory. Adding the "-r" option synchronizes the directory recursively. 

rsync -vtr /home/aditi/readings shyam@127.0.0.1:~/readings

Example 4: Synchronize all contents of Shyam's "music" directory to the music directory in present working directory. 

rsync -vtr shyam@127.0.0.1:~/music music

Example 5: Synchronize all music from Shyam's "music" directory to Ram's "music" directory. Using the "-a" option triggers the "archive" mode i.e. it synchronizes not only the content but also permissions, attributes, timestamps, ownerships, etc. Using the "-z" option uses compression during transmission, speeding up the transfer. 

rsync -az shyam@127.0.0.1:~/music ram@10.9.12.110:~/music

For more details on any of the commands discussed here, please refer to the manpage. 

man rsync
